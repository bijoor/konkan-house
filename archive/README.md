# archive/

Legacy scripts and companion docs kept for reference only.

- `render_with_blenderkit.py`, `render_with_procedural.py`, `render_with_textures.py`, `render_all_perspectives_with_textures.py`, `render_final_textures.py`, `render_realistic_perspectives.py`, `build_and_render_realistic.py` — one-off Blender rendering experiments that layered materials/textures onto the procedurally-built model.
- `apply_realistic_materials.py`, `apply_materials.sh` — pair for applying a fixed material scheme to `house-model.blend`. Paths inside the script are still relative to the repo root; move them back up if you want to re-run.
- `BLENDERKIT_INSTRUCTIONS.md`, `TEXTURE_DOWNLOAD_INSTRUCTIONS.md` — companion notes to the above.
- `test_crop_debug.py`, `test_texture_scale.py` — ad-hoc debug scripts.
- `konkan_house_lib_old.py` — the pre-refactor monolithic library. Superseded by `blender_3d.py`, `svg_2d.py` and the thin `konkan_house_lib.py` facade.

## Deprecated SVG generators (superseded by the TypeScript port)

Every 2D SVG the pipeline produces is now generated by the editor's TypeScript port under `editor/src/svg2d/`, run via `./scripts/regen_svgs.sh` (which invokes `npm run dump-svgs`). The parity harness (`npm run parity-all`) verifies each output is byte-identical to what these Python scripts would emit — currently **67/67 checks pass** across floor plans (3), elevations (5), roof (15), pillars (10), plus 34 primitive checks.

These originals still work — they import from `python/` and chdir to the repo root — so they can be re-run to regenerate the parity reference if the TypeScript port ever regresses:

- `regenerate_combined_svgs.py` — floor plans + elevations + roof (13 SVGs + master + manifest)
- `generate_floor_plans.py` — per-floor SVGs
- `generate_elevations_debug.py` — per-view elevation SVGs
- `generate_pillar_elevations.py` — 4 outer pillar elevations + N internal sections
- `regenerate_all.sh` — wrapper that ran all of the above sequentially

To regenerate the parity reference:
```bash
python3 archive/regenerate_combined_svgs.py
python3 archive/generate_pillar_elevations.py
```

None of these are on the active build/render path. Do not import from `archive/`.
